# HR Evidence

Этот файл нужен для проверки фактов о кандидате. Если факт не указан как подтвержденный, бот должен отвечать, что в материалах это не указано.

## Positioning

Статус подтверждения: подтверждено материалами портфолио.

Ярослав Сулейменов позиционируется как Business Analyst / Product-oriented Business Analyst / System Analyst.

Подтвержденные сильные стороны:
- структурирование требований, пользовательских сценариев, ролей, ограничений и критериев готовности;
- product thinking: связь scope MVP с бизнес-ценностью и проверкой гипотез;
- system analysis: сущности, статусы, lifecycle, data contracts, validation rules, API expectations;
- data analytics and reporting: SQL, Excel, Power BI, KPI analysis, dashboards, data quality checks;
- техническая грамотность для MVP: Python, aiogram, Google Apps Script, Google Sheets, React/Vite, Docker, GitHub Pages, Google Cloud VM;
- B2B sales background: понимание потребностей клиента, коммерческой ценности, маржинальности, влияния решения на сделку и операционный результат.

## Requirements

Статус подтверждения: подтверждено материалами портфолио.

Можно утверждать:
- кандидат умеет выявлять и структурировать требования;
- описывает user stories, use cases, acceptance criteria, MVP scope, ограничения и риски;
- готовит developer handoff и business handoff;
- связывает требования с бизнес-целью и основным пользовательским сценарием.

Доказательства:
- SmartQuote / Zaberman Calculator;
- Warehouse Control / Loading Control;
- Client Order Tracking MVP;
- Family Menu;
- AI Portfolio Assistant.

## Developer Collaboration

Статус подтверждения: подтверждено на уровне handoff, спецификаций и постановки задач. Размеры команд не указаны.

Можно утверждать:
- кандидат готовит материалы для разработки: architecture docs, data contracts, test cases, UAT cases, setup guides, implementation plans;
- описывает сущности, статусы, flows, edge cases и validation rules;
- умеет переводить бизнес-логику в формат, понятный разработке.

Нельзя утверждать без дополнительных данных:
- размер команды;
- роль тимлида;
- управление командой разработки;
- production ownership в команде 5+ человек.

## SQL And Data

Статус подтверждения: SQL указан как навык в Data Analytics & Reporting. Портфолио подтверждает data analytics/reporting literacy, но не подтверждает production SQL ownership в конкретном проекте AI Portfolio Assistant.

Можно утверждать:
- SQL входит в набор аналитических навыков кандидата;
- кандидат работает с KPI, dashboards, operational reporting, data quality audit, расчетными таблицами и аналитическими кейсами.

Нельзя утверждать:
- SQL является стеком AI Portfolio Assistant;
- кандидат писал production SQL для конкретного проекта, если это прямо не указано в контексте.

## API And Integrations

Статус подтверждения: подтверждено на уровне MVP, прототипов и integration boundaries.

Подтвержденные примеры:
- AI Portfolio Assistant использует DeepSeek/OpenAI-compatible API;
- Family Menu и Client Order Tracking используют Google Apps Script / Google Sheets как быстрый backend;
- SmartQuote и Warehouse Control содержат API expectations, data flows, Google Apps Script / Sheets или planned integration boundaries.

Ограничение:
не заявлять enterprise integration ownership или сложную production architecture, если это прямо не указано.
